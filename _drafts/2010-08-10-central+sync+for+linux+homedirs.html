--- layout: post title: central sync for linux homedirs date: '2010-08-10T15:53:00.005Z' author: Tim Abell tags: - linux modified_time: '2010-08-10T16:22:35.305Z' blogger_id: tag:blogger.com,1999:blog-5082828566240519947.post-7808326969744360361 --- On the subject of central synchronisation for Linux desktops.

The standard way of keeping centralised copies of home directories for linux desktops is as far as I can gather to mount them via NFS,

For desktops that are always connected to the network where there is a file server that is always available, this is a perfectly reasonable option. However for laptops which sometimes will and sometimes won't be connected, a different solution is needed.

I've spent a lot of time trying out options and thinking about this, and currently I find no options that do the job to my complete satisfaction.

The nicest solution I have found so far is using unison to synchronise manually.

It is difficult to set up linux to synchronise the user's home directory when they log in & out.

Rsync is good for background sync.

Unison is a great tool, but requires a large amount of setup to get everything right.

Something that windows does that linux does not currently do is have a separation in profiles between things that should be synchronised and things that shouldn't. This largely manifests itself as a "Local Settings" (i.e. don't synchronise) folder in the user's profile that will not by default be copied to/from the file server when the user logs in & out.

More on active directory & linux: http://reverendted.wordpress.com/2006/09/12/linux-goes-mad/

So what needs to be done?

Maybe we need some more work to be done on the server side of linux for managing desktops & laptops.

Perhaps we need a standard to be set that allows applications to explain to a synchronisation system what should and shouldn't be synchronised, and maybe even when it is safe to run a synchronisation.

Something like a .local folder, or a text file that states which files/folders should and shouldn't be synchronised.

Applications then need to follow this standard and make sure things like cached data are not included in synchronisation.

We also need user friendly setup of this synchronisation to a central file server.
